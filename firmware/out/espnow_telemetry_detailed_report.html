Executive Summary

Confirmed The serial capture contains raw ESP-NOW telemetry frames with Espressif OUI 7f:18:fe:34. All decoded telemetry frames are carried by ExpressLRS MSP_ELRS_BACKPACK_CRSF_TLM (0x0011) except three Betaflight MSP 0x0059 VTX-config fragments.

18 telemetry values decoded This capture includes exactly the main 18 EdgeTX-visible CRSF values you expected: 10 link stats, 4 battery stats, 3 attitude axes, and 1 flight mode string.

Link stats origin The high-count 0x14 Link Statistics frames are best explained as ExpressLRS link-health telemetry generated by the TX/RX link layer, not as a Betaflight sensor frame scheduled at the same rate as battery or attitude. The measured cadence is approximately 4.14 Hz, matching the ExpressLRS TLM_REPORT_INTERVAL_MS default around 240 ms.

Battery note Battery voltage is present. It ranges from 16.8 V to 18.7 V. The first battery remaining sample is 0%, then subsequent samples are 44% down to 36%, so the initial remaining-percent byte may be a startup/invalid value.

Protocol References Used

  • ExpressLRS: MSP_ELRS_BACKPACK_CRSF_TLM = 0x11 from src/lib/MSP/msptypes.h.
  • ExpressLRS TX link stats path: LinkStatsFromOta() updates RSSI/LQ/SNR from OTA link-stat packets, then checkSendLinkStatsToHandset() creates CRSF_FRAMETYPE_LINK_STATISTICS with makeLinkStatisticsPacket(), delivers it to the handset, and forwards it to the backpack with sendCRSFTelemetryToBackpack().
  • ExpressLRS RX link stats path: the RX firmware sends link stats to the flight controller at SEND_LINK_STATS_TO_FC_INTERVAL = 100 ms, using the same CRSF link-stat frame type when connected.
  • EdgeTX: CRSF IDs and sensor mapping from radio/src/telemetry/crossfire.h/cpp: Battery 0x08, Link Statistics 0x14, Attitude 0x1E, Flight Mode 0x21.
  • Betaflight: src/main/telemetry/crsf.c schedules battery, attitude, flight mode, GPS/baro/vario/heartbeat frames. Its receive-side CRSF code handles 0x14 Link Statistics for RSSI/link health input, but the normal telemetry schedule does not list Link Statistics as one of the sensor frames it emits.
  • Numeric decode follows EdgeTX byte offsets and big-endian signed/unsigned conversion. Battery voltage/current use precision 1, attitude is reported here as radians using EdgeTX's raw / 10000 display scale.

Why Link Statistics Dominates

Short answer: the many 0x14 frames are link-health heartbeats from ExpressLRS, not Betaflight repeatedly sending the same sensor group.

The capture has 135 Link Statistics frames over the 32.6 s active ESP-NOW window: 4.14 Hz. ExpressLRS exposes TLM_REPORT_INTERVAL_MS for TX-module-to-handset LinkStats reporting, with a documented default around 240 ms, which is 4.17 Hz. That lines up with the capture almost exactly.

ExpressLRS link stats route

RX RF link measures uplink RSSI/LQ/SNR and embeds link stats into OTA telemetry.
TX receives OTA packets and LinkStatsFromOta() updates linkStats.
checkSendLinkStatsToHandset() periodically emits CRSF 0x14 using makeLinkStatisticsPacket().
The same frame is also sent to the backpack via sendCRSFTelemetryToBackpack(), then appears in this ESP-NOW capture.

Betaflight telemetry route

Betaflight CRSF scheduler sends FC sensor frames such as Battery 0x08, Attitude 0x1E, Flight Mode 0x21, GPS/baro/vario when enabled.
The scheduler runs on a 100 ms cycle and spreads enabled sensor frames across that cycle.
Betaflight parses incoming 0x14 Link Statistics for RSSI/link health, but normal sensor scheduling does not produce the high-rate 0x14 stream seen here.
